WEB UI PROTOTYPE

Интерфейс для файла, prompt и ответа от внешних сервисов.

Лендинг, авторизация, чат-подобная рабочая область и история запросов в одном минималистичном интерфейсе. Сервер рендерит страницы, а точки интеграции с авторизацией, `сервисом а` и `сервисом б` уже выделены.

{% if session %} Перейти в чат {% else %} Перейти к авторизации {% endif %} Как это работает
mock pipeline
  1. Пользователь проходит вход и получает cookie с токенами.
  2. На рабочей странице загружает файл и пишет prompt.
  3. UI отправляет запрос в `сервис а`.
  4. Ожидает JSON-ответ от `сервиса б` через long polling или websocket.
  5. История диалогов остается доступной в sidebar и на отдельной странице.

01

Авторизация

Восстановление сессии по cookie, обновление access token через refresh token и редирект на вход при невалидной сессии.

02

Рабочая область

Чатовый интерфейс с историей слева, окном переписки справа, прикреплением файла и отправкой prompt.

03

История

Отдельная страница со всеми диалогами и быстрым переходом к нужному чату без лишней навигации.
